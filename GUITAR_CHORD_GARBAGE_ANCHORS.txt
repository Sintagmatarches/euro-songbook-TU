Guitar chord / tab garbage anchors
==================================

Purpose
-------
This file is the approval list for finding and removing guitar-service garbage from song texts.
The target garbage is not "any strange word"; it is guitar chords, tabs, fingering lines,
transposition notes, capo/barre notes, and chord-section headers imported from source sites.

High-confidence line patterns
-----------------------------
Use these as sanitizer regexes after candidate selection.

1. Guitar tab string lines:
   Lines beginning with a guitar string name or number, followed by a pipe and tab symbols.
   Examples:
   E|---0-1-3---
   B|---3-3-4---
   G|-----------
   D|-----------
   A|-----------
   e|-----------
   5|-2-2--2--------|
   6|--------3-3--3-|

   Pattern idea:
   ^\s*(?:[eBGDAE]|[1-6])\|[-0-9hpsx/\\| ]{6,}\s*$

2. Numeric tab lines without a pipe:
   Examples:
   5 -----6-6-5-5-3--------6-6-5-5-3-----
   6 -3-3-----------3--3-3---------------

   Pattern idea:
   ^\s*[1-6]\s+[-0-9hpsx/\\ ]{8,}\s*$

3. Plain chord lines:
   Examples:
   F#m Em            C    Edim               Hm
   F#sus F#7   Bm
   C G Am F

   Pattern idea:
   chord token = [A-H](#|b)?(maj|min|m|dim|aug|sus|add)?[0-9]*(/[A-H](#|b)?)?
   remove a line if it contains only 3+ chord tokens and separators/spaces.

4. Inline bracket chords:
   Examples:
   [Am]text [F]text [C]text

   Pattern idea:
   remove bracket tokens like \[(chord token)\], but keep the lyric text around them.

5. Chord-section headers and notes:
   Remove lines containing approved anchors such as:
   Акорди (варіант 2):
   Акорди у тональності G (за джерелом 1):
   Зміна тональності: C#
   Краще звучить, якщо усі акорди грати з баре
   Простіше буде зіграти, якщо використати такі акорди:

Important false positives
-------------------------
Do not remove these only because they contain similar substrings:

- German "Tabak", "Kautabak", "Tabakswolkenduft" are normal lyric words.
- German "Stab", "Zauberstab" are normal lyric words.
- German "griff", "ergriff", "trifft" are normal lyric words.
- German "Riff" can be a normal word meaning reef in lyrics.
- Ukrainian/Russian "акордом" can be a lyric word, not a chord header.
- Alphabet songs like "A, B, C, D" are not guitar chords unless surrounded by guitar context.

Efficient cleanup strategy
--------------------------

Do not browse every page or fetch every song through the public API.

1. Candidate selection in D1:
   Use SQL to select only records whose lyrics contain strong anchors or tab-looking lines.
   This should reduce the work from all songs to a small candidate set.

   Candidate LIKE anchors for lyrics and song_versions.lyrics:
   - %Акорди%
   - %акорди%
   - %Аккорды%
   - %аккорды%
   - %тональності%
   - %тональность%
   - %каподастр%
   - %капо%
   - %баре%
   - %перебір%
   - %перебор%
   - %chords%
   - %tabs%
   - %capo%
   - %transpose%
   - %barre%
   - %E|%
   - %B|%
   - %G|%
   - %D|%
   - %A|%
   - %e|%
   - %5|%
   - %6|%

2. Dry-run report:
   For each candidate, run sanitizer in dry-run mode and output:
   - song id
   - title
   - country
   - source
   - removed line count
   - removed line samples
   - before/after text length

3. Human review:
   Review only the dry-run report, not the full database.
   Add new anchors here if the report reveals another recurring pattern.

4. Apply cleanup:
   Update only changed rows:
   - songs.lyrics
   - songs.verified_translation if needed
   - song_versions.lyrics

5. Rebuild/search sync:
   After DB cleanup, rebuild the search auxiliary/index if this project requires it.

6. Deploy:
   If code/scripts are changed, deploy main production after verification.

Proposed implementation shape
-----------------------------

Create a script such as:
scripts/clean_guitar_chord_garbage_remote.mjs

Modes:
- --dry-run
- --apply

The script should:
- query candidates by SQL anchors;
- sanitize line-by-line;
- write a JSONL/CSV dry-run report;
- update only records where sanitized text differs;
- preserve audit output so rollback is possible.

Root-cause fix
--------------
After cleaning existing data, the same sanitizer must be applied in import/repair paths,
especially pisni.org.ua import and any generic HTML parser/importer that can ingest chord blocks.
Otherwise the garbage will return on the next import.

False positives discovered during cross-language inspection
----------------------------------------------------------

Do not use these as deletion anchors by themselves:

- "аккорды" / "акорд" as a normal lyric word:
  Examples include "Позабытой сонаты аккорды", "Так берет аккорды он",
  "Аккорды великой всесильной любви". These are lyric lines unless the line is a
  standalone header like "Аккорды" or appears near chord/tab structure.

- "акорди" as a normal lyric word:
  Keep lines like "Щоб летіла акордом вона" unless they are section headers or
  accompanied by chord/tab lines.

- "баре" in Russian:
  Often means "in a bar" or "masters/lords", not guitar barre.
  Examples: "Сидим мы в баре...", "Гой! вы сгиньте, баре знатные."

- "barre" in French:
  Usually means helm/bar/rail in sea songs, not guitar barre.
  Examples: "prends la barre", "tenir la barre".

- "capo" in Italian:
  Often means head/chief, not capo device.

- "guitar", "gitarre", "gitara":
  Often normal lyric content about a guitar. Do not delete unless the line is an
  instruction/header or tab/chord structure.

- "transpose":
  Can be a normal English verb in lyrics. Do not delete without music-service context.

- Short programming words like "var":
  Too noisy in Scandinavian/Romance lyrics. Do not use standalone.

Revised deletion rule
---------------------

Candidate search may use broad anchors, but deletion should be conservative:

1. Always delete complete tab lines:
   - E|..., B|..., G|..., D|..., A|..., e|...
   - 1|..., 2|..., 3|..., 4|..., 5|..., 6|...
   - numeric tab lines like "5 -----6-6-5-5-3-----"

2. Delete chord-only lines:
   - lines made almost entirely of 3+ chord tokens such as "F#m Em C Edim Hm";
   - allow separators/spaces, but no normal lyric words.

3. Delete explicit chord-section headers:
   - "Акорди:"
   - "Акорди (варіант 1):"
   - "Акорди (варіант 2, в тональності \"Am\"):"
   - "Аккорды" only when the whole line is a header.
   - "Chords:", "Acordes:", "Accordi:", "Akkorde:", "Akordy:".

4. Delete explicit guitar-service instruction lines:
   - "Акорди: капо на 4 ладі"
   - "Простіші акорди, з капо на 3 ладі"
   - "Краще звучить, якщо усі акорди грати з баре"
   - "Зміна тональності: C#"
   - "Оригінальна тональність"
   - "Інші тональності"

5. Do not delete broad musical words unless they match one of the above structures.

Additional deletion rules for non-chord residue
-----------------------------------------------

1. HTML entities:
   Replace harmless entities inside lyrics instead of deleting the line:
   - &nbsp; -> normal space
   - &laquo; -> «
   - &raquo; -> »
   - &amp; -> &
   This is normalization, not line deletion.

2. URL/source lines:
   Do not delete every line containing a URL automatically. Some source notes may be useful.
   For cleanup candidates, report lines containing:
   - http://
   - https://
   - www.
   - "Quelle:"
   - "Wersja z"
   - "Więcej:"
   - "nach :"
   The dry-run report must show these for manual approval.

3. Unsafe source/UI anchors:
   Do not use "register" as a cleanup anchor.

Meaning:
- &nbsp; is a non-breaking space. It should become a normal space.
- &laquo; is «.
- &raquo; is ».

This class should be normalized in place, not removed line-by-line.

Replacement rules to approve:
- &nbsp; -> regular space
- &laquo; -> «
- &raquo; -> »
- &amp; -> &
- &quot; -> "
- &apos; -> '
- &#NNN; -> decode numeric HTML entity
- &#xHH; -> decode hex HTML entity

Inspection date: 2026-04-29.
Method:
- full public pass over Estonian language catalog;
- checked 292 Estonian songs;
- found many lines with the notation :,: ... :,:.

Meaning to approve:
The notation appears to mark repeated text, usually "repeat twice". It should eventually be
represented structurally, but until the UI supports repeat metadata, it should be normalized
consistently rather than treated as garbage deletion.

Important forms found:

   :,: Prii olen ma! :,:

   :,: me röövisime rikkaid ja vaeseid

   Refr. :,: Läks aga metsa mängima,
   läks aga laande lauluga. :,:

4. Repeat notation embedded inside a line:

5. Shortened repeat:
   Refr. :,: Läks aga jne. :,:
   Refr. :,: Ei paremat .... :,:

Proposed temporary normalization options for Estonian repeats
------------------------------------------------------------

Option A: Preserve notation but standardize spacing.
Examples:
- ":,:text:,:" -> ":,: text :,:"
- "Refr. :,:text" -> "Refr. :,: text"

Pros:
- Low risk.
- Does not invent UI behavior.
- Keeps source meaning.

Option B: Convert to explicit text marker.
Examples:
- ":,: Prii olen ma! :,:" -> "Prii olen ma! [повторить 2 раза]"
- multi-line block gets one marker after the closing ":,:"

Pros:
- Easier for users to understand.
Cons:
- Adds Russian UI text inside Estonian lyrics.
- Not ideal for multilingual site.

Option C: Future structured representation.
Store repeat spans in metadata, render with UI.

Pros:
- Correct long-term model.
Cons:
- Requires product/UI work.

Recommended for now:
Do not remove :,: notation as garbage.
For current cleanup, only standardize spacing and report all repeat blocks in dry-run.
Later, implement a structured repeat marker.

Current approved cleanup semantics after repeat UI implementation
-----------------------------------------------------------------

The site now supports a visual repeat marker. Therefore repeat notation is no longer a
"future" cleanup item. It must be converted to structured repeat rendering, not deleted.

Do not delete these repeat markers as garbage:
- :,: ... :,:
- x2
- ×2
- 2 раза
- 2 рази
- два раза
- twice
- repeat / repeats / repeated
- повторить
- повторяется
- повторится
- повторити
- повторюється
- повториться
- kordub
- korrata
- last line repeats
- previous line repeats
- последняя строка повторяется
- последняя строка повторится
- последнюю строку повторить
- остання строка повторюється
- останній рядок повторюється

Approved behavior:
1. :,: opening and closing markers define a repeated span.
   - Remove the literal :,: markers from visible lyrics.
   - Render the affected line/block with the right-side repeat bar and x2 marker.
   - Preserve normal colons inside lyrics, e.g. "nagi konn : on ..." or "sõnas hiir:".

2. Chorus/refrain and repeat are separate entities.
   - "Припев. (2 раза)" means chorus + repeat x2.
   - "Refr. :,: ... :,:" means refrain/chorus + repeat x2.
   - Do not convert chorus into repeat or repeat into chorus.
   - If both are present, show chorus marker on the left and repeat marker on the right.

3. Textual repeat instructions should become structured repeat metadata/rendering.
   Examples:
   - "Припев. (2 раза)" -> chorus block with repeatCount = 2.
   - "Chorus x2" -> chorus block with repeatCount = 2.
   - "last line repeats" -> previous lyric line gets repeatCount = 2; the instruction line is removed.
   - "последняя строка повторяется" -> previous lyric line gets repeatCount = 2; the instruction line is removed.

4. Repeat markers are not deletion anchors.
   They are semantic markup. Deleting them without adding repeat metadata loses song structure.

5. Cleanup output should distinguish actions:
   - delete: guitar tabs, chord-only lines, chord-section headers, guitar instructions.
   - normalize: HTML entities and spacing.
   - convert to structure: repeat markers and textual repeat instructions.
   - review manually: source notes, URL notes, explanatory notes.

Current approved replacement/normalization map
----------------------------------------------

HTML entities:
- &nbsp; -> regular space
- &laquo; -> «
- &raquo; -> »
- &quot; -> "
- &apos; -> '
- &#NNN; -> decoded numeric Unicode character
- &#xHH; -> decoded hexadecimal Unicode character

Spacing/punctuation around entities:
- After entity decoding, collapse repeated spaces.
- Convert non-breaking spaces to normal spaces.
- Do not delete punctuation just because it was adjacent to an entity.

Repeat notation:
- :,: text :,: -> text rendered as repeat x2.
- Refr. :,: text :,: -> chorus/refrain rendered as repeat x2.
- x2 / ×2 / 2 раза / 2 рази / два раза / twice near a section marker -> repeatCount = 2.
- "last line repeats" and equivalents -> previous line rendered as repeat x2.

Explicitly unsafe anchors / do not use globally
-----------------------------------------------

- "бой:"
- "бій:"

These are not safe cleanup anchors because war/military themed songs can contain these words
as real lyrics or headings. They must not be used for automatic deletion.
