Repository inspection was performed to identify the actual technology stack, routing structure, API surface, authentication model, role model, and database schema. Automated validation was then executed using the repository's existing Node-based test suite, Python `unittest` suite, and the Cloudflare Pages functions build command.

Endpoint validation was performed against a locally started Cloudflare Pages development server backed by local D1 bindings. Executed API scenarios covered catalogue retrieval, title and lyrics search, filter combinations, song-card retrieval, authentication flows, protected-action denial, favourites, comments, request submission, fragment-report submission, draft creation, draft collaboration, and selected timing measurements. Browser scenario checks were executed in headless Microsoft Edge to confirm that the SPA loaded and that the song page rendered version controls, comparison controls, comments, links, and AI prompt buttons. Results were mapped to thesis requirements and research questions using only executed evidence.
